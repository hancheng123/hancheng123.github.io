vim编辑器快速上手

将esc和capslk键切换

为什么要切换呢？因为在vim的使用中esc会频繁使用到，但平时距离太远，并且capslk大小写切换键不常用，切换后能提高我们在vim的编辑速度。
首先打开windows运行cmd,打开命令框后
输入:regedit
打开注册表,按照这个目录点开

   [HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\Keyboard Layout]

 在keyboard Layout文件下新建多字符文件
 修改文件名后（随意写一个文件名）
 双击文件打开将下面的代码粘贴到内容上

“Scancode Map”=hex:00,00,00,00,00,00,00,00,03,00,00,00,3a,00,01,00,01,00,3a,00,00,00,0 0,00
然后重启电脑就行了。

个人博客搭建

总体流程

博客搭建可以说简单也可以说难，对于小白来说一切都是陌生的，当出现一些问题时很难
自己解决，这个时候就需要反复进行操作，在不断的失败中总结经验并且了解每一个步骤
的意义。
首先我们需要安装nodejs和git ,先安装nodejs，不管是hexo还是git都是在有node的前提
下运行的。等node和git安装完后就可以安装博客静态模板Hexo,先在本地操作，没问题后
再部署到github托管，这里是需要安装一个deployer.git的插件后面会说到。

安装node和git

第一步 node安装包的地址：https://nodejs.org/en/
选择LTS版本安装就行了，也就是左边的版本，下载后进行安装，一路next,不需要修改
安装路径，全按默认就好。
打开windows或Linux运行，输入cmd,打开命令框
输入 node -v      //查看node版本
输入 npm -v       //查看npm版本
1.输入 npm install -g cnpm --registry=https://registry.npm.taobao.org
//因为国内的npm包管理器比较慢，选择用npm安装淘宝镜像源cnpm，会自动安装
输入 cnpm -v      //查看cnpm
2.输入 cnpm install -g hexo-cli  //安装hexo静态博客模板，等待下载
输入 hexo -v      //验证是否安装
3.输入 mkdir blog   //在当前根目录下新建一个blog文件夹
输入 cd blog      //查看当前文件夹路径
结束cmd，记住blog文件夹的位置，之后的步骤都在blog文件夹下进行，整个博客都安装
在blog文件夹下，如果出现解决不了的问可以直接删除blog文件，重新搭建，比较方便

第二步 安装git
git安装包地址：https://nodejs.org/en/
安装包下载完后打开，也是默认安装，不要改默认安装途径。一路next,一切按照默认来就行了。
git安装完后，在文件右键就会有git bash here 的选项那就恭喜安装成功了。

hexo搭建

安装完git后就能进行之后的操作了，首先找到之前新建的blog文件
打开空文件夹,右键，点击git bash here,打开命令框
1.输入： hexo init    //在blog文件下初始化Hexo,等待初始化。。。
这时已经生成了hexo的初始页面，接下来将hexo从本地部署到github托管，注册github
账号，新建一个仓库，设置仓库名为: 你的ID.github.io.git，之后可以通过
https://github.com/仓库名  访问我们的hexo博客
2.输入：hexo s        //系统会给你个地址，可以在本地访问，有的访问不了，没关
系，可能是防火墙的关系，这一步只是用来验证是否初始化成功，可省略
3.cnpm install --save hexo-deployer-git  //安装部署插件，部署到的远端插件
4.修改blog文件下的_config.yml文件
确认在blog目录下       //退回上一级 cd ..   退回俩级 cd ../..
vim _config.yml       //进入vim编辑器，找到type:  修改并在添加repo和branch
type: git
repo：https://github.com/ID/ID.github.io.git
branch: master
按esc :wq退出编辑器

4.hexo clean          //清楚之前的public文件，punlik文件存放所有部署的文件
就是将public文件的内容部署到github仓库去，有自己域名的可以将public文件内容复
制到nginx环境下就行了。
5.hexo g              //生成新的public文件，这时可以hexo s 在本地验证一下
6.hexo d              //部署到仓库，这时会生成一个.deployer.git文件，用来存
放部署的文件，每次部署都将public文件内容copy过来，所以要将旧的public文件
hexo clean，就是为了避免冲突。
这时就完成了基础的搭建，可以在网页地址栏输入：https://你的ID.github.io访问
搭建好的个人博客了。

写文章

打开bolg文件，右键git bash here打开命令框
1.cd source/_posts    //寻址到存放文章的文件夹，不然会将文章直接创建在blog下
2.hexo n "文章名称"   //创建 文章名称.md 文件，注意空格
3.vim 文章名称.md 文件 //进入vim编辑器，在vim编辑器下写文章，注意文章的.md后缀不要忘了。vim的使用后面的
的文章会有详细的说明。同学们可以专门学一学vim编辑器的用法，vim是个神器。
暂时知道按a或i进入插入模式能进行编辑就行，esc回到普通模式

写好文章后，按esc+:+wq保存并退出编辑器
4.hexo s          //验证
5.hexo g              //生成新的public文件
6.hexo d              //部署到github仓库
在自己网站搭建博客的小伙伴将新生成的pubilc文件的内容复制，通过xftp连接服务器
传输文件。将文件复制到服务器下的nignx文件夹下就行了。

修改博客主题样式

1.先下载一个主题文件
输入： git clone https://github.com/litten/hexo-theme-yilia.git themes/yilia
 //将别人发表的主题文件下载到我们的 themes/yilia文件下，（推荐以上的样式）
 输入： cd themes
        ls -l          //查看是否生成了yilia主题文件，themes保存着主题文件
2.修改_config.yml文件
确保在blog文件目录下
输入： vim _config,yml  //进入编辑 找到theme: 修改
theme： yilia           //初始主题文件文lanscape,将其改为新下载的主题文件
esc :wq 退出编辑器回到命令框
3.hexo clean
4.hexo g
5.hexo d                //完成新主题的部署
如果对主题不满意，想自己更改的，学过HTNL/css,js的，应该就很容易了，毕竟这是
个静态博客，和静态网站大同小异。

总结

按照本文章的流程，肯定能创建一个完整得到hexo博客，如果出现无法部署的问题，
可能是换主题是出现可能是新主题的问题导致，如果出现生成新public文件里面
index.html文件为0kb的情况，也是新主题导致的，请更加仔细的分析问题，寻找出
错的地方，希望你能成功，不要放弃

怎么用git

第一章 git的基础操作

安装git:Git下载地址： https://git-scm.com
安装过程跟着程序走就行，不要修改默认地址
安装完成之后先进行本地操作：
  先新建一个文件夹，在文件夹里面右键点击git bash here打开命令框。我们在哪个
文件夹打开gitbash就会在当前的根目录中，可以说是很方便了。
在编辑框中输入：
1.git init
(在本地创建一个git仓库：仓库具有保存我们历来提交文件版本的功能，也具有备份的功能。）
创建好git仓库是可能会有一个git文件夹，这是git仓库的初始化文件，不用去管。
接下来我们可以在先pwd确保在当前目录。

2.比如创建一个study.html文件
我们可以输入
git status //查看库的状态，是否有文件处于未暂存或未提交状态：保存到git库有三
个步骤：1.文件单方面保存  2.将文件存入暂存区 3.将暂存区的文件提交到仓库

3.git add .
1.指定文件存入暂存区
git add study,html  //指定将study文件存入暂存区
2.将全部文件存入暂存区
git add .  //将目录下的文件全部加入暂存区

将文件加入暂存区两种方法都可以，如果文件过多就全部加入暂存区

4.git commit -m '注释'
将缓存区的文件提交到仓库，“注释”可写可不写，主要用于提示提交文件的备注，便于
以后的查看，建议养好注释的习惯

如果要修改文件，就重复3,4步

5.git log
查看提交的文件，会显示unknown人员提交的文件
可以写
git config --global user.name '姓名' //回车继续输入
git config --global user.email  邮箱 //回车
这样再提交，提交文件就会显示该文件所提交人的信息，有助于管理。

以上就是在gitbash的本地基本操作，再补充一些更灵活的操作方式。

   1.如果想回到之前保存的文件状态
   输入
   git checkout ID //ID可以通过git log 获取，gitlog会显示每次提交的文件ID，
   高亮黄色的就是。
   git checkout HEAD^ //回到上次提交的状态
   注意回到之前的版本，之后的版本会消失

   2.有的文件万年不变，不需要对其进行提交，将其忽略：
   在根目录下创建一个.gitignore 文本,编辑文本
    /   /  这个是需要忽略文件的目录
    /   /
    projiect.properties

   3.如果文件修改过后还未提交仓库想查看更改的内容
   git diff
   会将你更改的部分显示出来，方便查看

   4.如果回退到之前的版本之后，又后悔了，想回到新保存的文件，俗称“后悔药”
   git reflog          //显示提交的每一次记录，并带有ID
   git checkout ID     //回到想要的版本并且git log 查看在仓库中也存在新版本

   5.当文件保存了，但还没加入暂存区，想要撤回修改的代码
   git diff //查看修改的代码和目录
   <strong> git checkout 文件根目录</strong>  //会将修改的代码全部删除，如果
   代码量不是很大，建议手动修改

   6.如果文件已经add 加入暂存区的，要撤回修改
   git reset HEAD                 //HEAD表示当前文件，HEAD^表示上个版本文件
   撤回之后再继续5.步骤

第二章 git分布式

在工作中，项目需要进行多线同时开发，这时就需要git分布式开发了
一个项目需要进行升级维护并且进行功能开发，升级作为master主体，开发作为branch
分支
1.在master主体下打开git bash
git checkout -b branch   //注意空格
git branch               //查看仓库的分支，这市显示master和branch

2.在branchf分支下就能新建文件开发了
如果要删除分支
git branch -d branch    //删除后分支的所有文件消失

3.新建的文件 继续基本 加入暂存和提交仓库
git add .
git commit -m '新分支'

4.当开发完成的需要和主体合并
必须先回到master
git checkout master     //回到主体
git merge branch -m     //将分支合并到主体，注意分支还是存在的
--也可以主体合并到分支，回到分支
git checkout branch
git merge master -m

第三章

前面的基础操作都是在本地进行的，接下来是将提交到远程库
远程库托管有俩种：github和gitlab
github适用于个人分享技术经验 保密不强
gitlab适用于公司项目托管分布开发，具有私密性

 github
需要注册一个账号    地址 https://github.com
需要创建一个仓库    仓库地址 zhanghao/zhanghao.gitub.io.git
1.首次clone项目
在根目录下打开git bash
git clone 仓库地址 //在根目录下会创建一个仓库名的文件夹，这个文件夹就是整个
仓库了，里面也有一个.git仓库初始化文件。
克隆完后以后的文件更新只要拉取下来就行了，不用整个仓库clone
git pull //拉取文件

2.进行开发，提交至github远程库
首次提交
 提交之前也是需要add commit 的基本操作的，只有本地先提交仓库了，才能直接提交     到远程库。
 1.    git remote add origin https://github.com/zhanghao/zhanghao.github.io.git     //可能会提示已经执行了，就直接下一步，就不用管了

 2.    git push -u origin master（弹出账号密码框）
 //输入github email 和密码
 这样就提交成功了，可以查看github仓库已经有提交的文件了

 以后提交直接
 git push 

  在github上还可以进行个人博客的托管，有兴趣的看之后的文章


 gitlab
 注册gitlab账号 
 创建新项目点开就有步骤操作
 要想在gitlab上clone仓库需要配置ssh秘钥，不然没法创建
 1.在根目录右键打开git bash
   配置账户，确认权限
   git config --global user.name ''
   git config --global user.email 邮箱

 2.生成ssh秘钥
 在gitlab点开个人账号的设置，在设置里点SSH秘钥，然后生成公开秘钥
 将秘钥写入gitbash
 ssh-keygen -o -t rsa -b 4096 -c '邮箱' //-c后面的是注释，也是标题
 会提示ssh生成的id._rsa文件的地址，找到id._rsa文件用记事本打开，ctrl+a,ctrl
 +c 全选复制，粘贴到gitlab的SSH秘钥上，标题自动生成也就是''注释，然后提交就
 行了。

 3.克隆仓库
 git clone 仓库地址  //如果是新建的仓库，在gitlab打开仓库，里面就附有
 如果需要覆盖就 y
 之后就回车
 创建完了。

 项目协作：
 1.gitlab 首次提交
 先add commit
 git push -u origin master
 以后直接
 git push

 2.首次获取更新
 git clone 仓库地址 
 以后
 git pull //拉取更新文件

 分支
 1.先拉取文件
 git pull

 2.创建分支
 git checkout -b branch

 3.add commit

 4.git push 会报错系统会提示怎么做
 git push --set-upstream origin branch //这是系统提示，可以直接打这个代码，
 或直接复制粘贴系统的提示。
 创建并提交到远程库后，就能在gitlab库上看到有master 和branch分支了
 mater和branch互不影响

 5.合并分支
 先切换到master
 git checkout master
 合并
 git merge branch
 这都是在本地操作，要在gitlab上看到合并必须提交远程库
 git push //合并后gitlab库上依然存在branch分支
 删除远程分支
 git push origin :branch //不建议
